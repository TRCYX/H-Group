### 禁止无意义提示 \{#no-nonsense-clues\}

- 通常，玩家只在无事可做时才会弃牌。但不要误解“无事可做”。
- 玩家<Em>**不允许**</Em>随意提示任何牌：
  - 玩家<Em>**只允许**</Em>对实际可打出的牌给出*出牌提示*。
  - 玩家<Em>**只允许**</Em>对*留牌原则*部分中列出的牌给出*留牌提示*。
- 如果玩家无法给出合规的*出牌提示*或*留牌提示*，则他们**必须弃牌**。

### 节奏提示 \{#tempo-clues\}

- 此外，玩家可能会试图**再次提示一张已经提示过的牌**，以使其立即打出。我们将这种提示称为*节奏提示（Tempo Clue）*。
- *节奏提示*是指没有获取新牌的*出牌提示*。
- *节奏提示*非常低效——团队使用两个提示来获取一张牌。由于*节奏提示*效果非常差，我们约定玩家**只能在特殊情况下执行节奏提示**，比如当玩家只能给出提示时。
- 例如在 3 人游戏中：
  - Alice 处于无法弃牌的特殊情况。
  - Bob 手中有一张 2，之前被*留 2 提示*过。他不知道这张 2 的颜色，但假设它是红 2（可打出）或蓝 2（不可打出）。
  - Alice 提示 Bob 红色，填补了红 2 的信息。这是一个*节奏提示*，因为没有引入任何新牌。（由于效率太低，除非 Alice 处于特殊的停顿状态下，否则她绝不会这样做。）
  - Bob 现在可以打出红 2。

### 提示价值下限原则 \{#minimum-clue-value-principle-mcvp\}

- 因此，如果玩家只能在给出*节奏提示*和弃牌中二选一，那么他们**必须弃牌**。
- 我们将这一概念正式表达为*提示价值下限原则（Minimum Clue Value Principle）*：**所有提示必须让至少一张之前没有提示过的牌打出或保留**。
- 换句话说，作为一个团队，我们自我要求很高，不会给出低效的提示。
